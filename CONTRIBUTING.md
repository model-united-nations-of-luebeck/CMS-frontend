# Contributing to CMS frontend

This conference management systen is a free and open project. We invite you to become a part of it and would love to receive your contributions.
There are multiple options for contributing.

## üêõ Bug Reports

When you've spotted something which doesn't quite work or is completely messed up, please [submit a bug report](https://github.com/model-united-nations-of-luebeck/CMS-frontend/issues/new/choose). We can only fix the bugs we know about. A detailed description helps us to reproduce the bug and fix it as soon as possible.

## üöÄ Feature Request

If you have an idea how we could make the conference management system even more awesome, feel free to share it by [creating an issue](https://github.com/model-united-nations-of-luebeck/CMS-frontend/issues/new/choose). Maybe we or someone from the community finds time to implement it.

## üíª Contributing Code

Of course also code contributions are welcome. To avoid any frustration on the developer and maintainer side, please stick to the following guidlines.

1. Create an issue first (if it doesn't exist yet) and assign yourself so that no one else is working on the same issue
2. You might have to fork this repo and clone it to be able to work on it.
3. Implement your changes according to coding best practices. Please test your code so that it works.
4. If it is your first time contributing to this repo, add your name to the `AUTHORS.md` file with your name and username. By doing this, you become on of the authors according to the license.
5. Create a pull request where you describe your changes an reference the issue(s) that this PR solves.
6. Wait until we come back to you. We will do a code review and if we are happy with the contribution, we will merge it. Otherwise we will give you feedback on how to improve your contribution.

Please reach out at any time in case you have questions or need support.
